Git上传github操作指南

发表于 2019-04-17 | 阅读次数：

因为我经常会忘了怎么上传…所以写个文章记录下来。本文仅记录如何使用git上传文件到github的操作，初版中不包括秘钥等设置，留在以后补充。其他情况不在本帖讨论范围

一、首先你得在github上创建一个新的仓库


打开github（当然是要先登录啦），进入之后在页面右上角点击“+”号，选择New repositoty（下图红框处）



然后在Repository name处填写你要新建的仓库名称（前缀系统以及默认给你填写你的用户名了）



上图还有其他选项，其中“Public”和“Private”用来选择是否公开这个仓库。最下面的initialize this repository with a README代表是否要生成一个readme文件让告诉大家这个是干啥的，这里的我们先不选择，输入完仓库名称之后戳右下角的create即可创建仓库。例如我现在创建一个test仓库，public，不创建readme文件。



到这一步github这边的设置就算结束了。



二、电脑安装git并且配置


安装过程这里不多做表述，安装完之后先进行git账号的配置。首先启动GitBash，可以在开始栏找，或者右键桌面选择“GitBash Here”，打开bash界面。



然后在上图黑乎乎的界面输入以下两条指令配置：


$ git config –global user.name “注册github的邮箱”


$ git config –global user.email “github的密码”


按照一般的教程接下来还要配置ssh，但我没有…以为懒（大家不要学我，这边我日后一定会补上的


接下来我们就进入了愉快的上传环节。上传流程为：在本地创建对应的缓存文件夹（即.git文件夹），然后将要上传的文件添加至该缓存文件夹，接着写一个提交声明，然后连接至github对应康库的位置，最后就是把缓存文件夹里的东西推到github就完事啦。听起来很复杂，我们直接看命令：


首先在要上传的文件夹或文件所在的文件夹打开bash，方法还是鼠标右键选择GitBash Here，然后输入（//后面内容为注释，不需要输入）：


git init  //即在当前位置创建一个.git文件夹作为一个缓存文件夹，如下图，我在test文件夹使用了该指令


然后使用：


$ git add .         //添加当前文件夹下的所有文件
$ git add .txt    //添加当前文件夹下的.txt这个文件 ,注意不能提交空文件夹，一定要有点东西，例如我在test下新建一个test.txt然后添加


接着提交一个提交说明


 git commit -m “说出你的故事”  //引号中的内容为对该文件的描述


然后关联github仓库，准备把.git文件夹里刚刚add的东西push到我们的github上


$ git remote add origin https://github.com/Wangggod/test.git 注意https字段后面的地址是你的仓库地址后面加.git

如果出现错误：fatal: remote origin already exists，则执行以下语句： 


$ git remote rm origin
再执行一次 $ git remote add origin https://github.com/Wangggod/test.git

最后执行：


$ git push origin master


如果出现错误failed to push som refs to…….，则执行以下语句，先把远程服务器github上面的文件拉先来，再push 上去。 


$ git pull origin master


总结起来的流程就是


git init


git add .


git commit -m”xxxxx”


git remote add origin https://xxxxxx


git push origin master


大概就这样，至于分支部分日后挖坑在写，还有其他的一些骚操作
